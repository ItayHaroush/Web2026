# מגבלות גודל העלאה (API / תמונות מבצע)

## סימפטומים

- **413 Payload Too Large** ביצירת/עדכון מבצע עם תמונה גדולה.
- הדפדפן עלול להציג גם **CORS** — לרוב זו תופעת לוואי: השרת (או nginx) חותך את הבקשה לפני Laravel ולא מחזיר כותרות CORS.

## מה לעשות בשרת (פרודקשן)

1. **Nginx** — להגדיל גוף בקשה, לדוגמה בתוך `server` או `location`:

   ```nginx
   client_max_body_size 25m;
   ```

2. **PHP** — `php.ini` (או `.user.ini` בתיקיית `public`):

   ```ini
   upload_max_filesize = 25M
   post_max_size = 26M
   ```

3. **הפעלה מחדששל PHP-FPM / nginx** אחרי שינוי.

## מה בקוד

- בפרונט: תמונת מבצע נדחסת אוטומטית לפני שליחה (מטרה: להישאר מתחת למגבלות nginx ברירת מחדל).
- ב־Laravel: ולידציה `max:12288` (KB) לתמונת מבצע — חייב להתאים ל־`post_max_size` בפועל.
